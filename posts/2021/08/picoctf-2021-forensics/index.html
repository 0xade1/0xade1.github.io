internals

We need to examine headers according to specifications:

We first have a 14 bytes File Header_ section with signature BM. Such signature is equivalent to BM support for Windows 3.1x, 95, NT, … etc and is always followed with a 40 bytes DIB header section.

File Header

offsetnamesize (bytes)hexvalue
0x0Signature2424dBM
0x2BMP size in bytes48e26 2c000x2c26e8 = 2893454
0xaoffset of pixel array (image data)4bad0 00000xd0ba = 53434

DIB Header - BITMAPINFOHEADER

A fixed size of 40 bytes (0x28) is expected for this header, but we have 53434, which is incorrect. Will have to patch this.

offsetnamesize (bytes)hexvalue
0xeHeader size4bad0 00000xd0ba = 53434

Patching:

echo "00000000: 424d 8e26 2c00 0000 0000 bad0 0000 2800" | xxd -r - tunn3l_v1s10n

After patching, we can open the BMP and read a fake flag “notaflag{sorry}”; so we have to investigate further

We better examine the DIB header more deeply to find any other inconsistencies. Since we have patched the DIB header length, the new header looks like this:

offsetnamesize (bytes)hexvalue
0xeHeader size42800 00000x28 = 40
0x12bitmap width (px)46e04 00000x46e = 1134
0x16bitmap height (px)43201 00000x132 = 306
0x1anumber of color planes (must be 1)201001
0x1cnumber of bits per pixel218000x18 = 24 bits = 3 bytes
0x1ecompression method40000 0000none
0x22size or raw bitmap data45826 2c000x2c2658 = 2893400

Rich of the above info, we now can figure out the different file sections with their size:

|<- 14 bytes ->|<- 40 bytes ->|<-------- 53434 - 40 - 14 = 53380 ------>|<----------- 2893454 - 53434 --------------->|
| File Header  | DIB Header   | Extra Bit Masks | Colour Palette | Gap1 | Pixel Array | Gap2 | ICC Color profile      |
|              |              |                 |                |      |<- 2893400 ->|<- 2893454 - 2893400 - 53434 ->|

The raw image is (1134 * 306) pixels of 24 bits (3 bytes) = 1041012 bytes. This is contradictory with the value of the raw bitmap data 2893400. It is likely that only a part of the image is being shown, so we may want to increase the number of pixels to match 2893400 // 3 = 964466 pixels.

We can patch to adjust the height to the value 964466 // 1134 = 850 = 0x0352 that can be represented with 5203 0000

echo "00000010: 0000 6e04 0000 5203 0000 0100 1800 0000" | xxd -r - tunn3l_v1s10n

Now the image can be open in Firefox and we see the flag picoCTF{qu1t3_a_v13w_2020}
